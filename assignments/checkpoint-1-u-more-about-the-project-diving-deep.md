**How to find a task to work on**

- Talk about the issue track structure. Does the project use tags to identify easy tasks? If yes, what tag(s) are used? How active are the issues? etc.

**Personnel and social structure**

- Who are the main contributors to the project (find out)?
- What is the governance model of the project (if any)? How do you know that?
- What communication channels are available, and what are their focuses? (Forums, mailing lists, issue tracker, IRC, Slack, etc.) Have you used any of them?

**Code structure and architecture:**

- How is the code structured? Do they use any frameworks? What is the stack used? What are the technologies they use (DB, Front-end, Queue Managers, frameworks)?
- What are the main modules? Does the project follow any high-level architecture or pattern?
- Figures/diagrams and descriptions are very welcome here!

**Setting up the workspace:**

- Are there any tutorials, guidelines, or FAQs on setting up the local workspace?
- Does the community recommend any specific operating system or IDE? Which ones?

**Code documentation:**

- Is there any code documentation available? Where?
- Is the architecture documented?
- Are there any coding standards, policies/guidelines available? Where?

**Submitting your contribution:**

- Are there guidelines on how to create a submission? Where?
- Is there any kind of checklist to follow?

**Use of DocGPT**

- If you used the Conversational Agent (Discord):
  - Explain how and which tasks you use the agent.
  - Where did it help your group with?
  - What did it not help you with
- If you did not use or stopped using:
  - Why did you make that decision and when?
