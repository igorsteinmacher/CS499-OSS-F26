# Lecture 0. Course Overview (Syllabus Day!)


## **About Open Source Software Development**

This course is intended to familiarize students with the fundamentals of Open Source Software development. We aim to prepare the students for the real world, exposing them to real projects. The practical objective of the course is to teach students how to set up a new project, foster participation, and get involved in OSS projects.

[Lecture 00 Slides](../assets/Slides/Lecture_00_Syllabus.pptx "Lecture_00_Syllabus.pptx")

### **Course Learning Objectives**

Upon successful completion of this course, students will be able to demonstrate the following competencies:

- LO1: Explain how a team interacts and collaborates to develop software in open source software.
- LO2: Differentiate between open source and closed-source software software;
- LO3: Use a version control system and ian ssue tracker as development tools;
- LO4: Evaluate and review code contributions;
- LO5: Explain intellectual property rights, and licensing of using open source;
- LO6: Participate in an OSS community, contributing code.
- [CS 599] LO7: Judge open source project management and onboarding mechanisms.
- [CS 599] LO8: Critique methods and mechanisms to manage OSS communities.

### **Expectations**

To complete this course successfully, please consider that you will need the following resources:

1. Canvas should be used on the current or previous major releases of Chrome, Firefox, Edge, or Safari browsers. Canvas runs on Windows, Mac, Linux, iOS, Android, or any other device with a modern web browser. Visit Canvas's technology requirements for more information.
2. A computer with an internet connection for class assignments.
3. A [git versioning control system](https://git-scm.com/) set up on your machine.
4. A programming language environment of your choice available on your machine (Python recommended)
5. A user account on [GitHub](https://github.com/).

**Effort:**

- You are expected to put an average of 10 hours/week of effort into this course (with some variations).
- There will be individual assignments (first half) and group assignments (second half)
- You will be required to contribute to an Open Source project, which will start in week 8.
  - Plan and work constantly. THIS IS TIME-CONSUMING, and leaving it to the last week may be problematic.
- Please check the Syllabus (Course Summary section) or the Course's calendar.
