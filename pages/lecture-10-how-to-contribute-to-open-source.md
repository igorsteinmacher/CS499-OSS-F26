You are requested to **contribute** to an OSS project. You will need to interact with the TAs, the community, and the existing code base. You MUST make a **code contribution**. You may start with a small contribution to documentation or translation to learn the process (although this is not required).

This semester, our target is contributing to the **R data.table** project.

- **[R data.table](https://github.com/Rdatatable/data.table)** - One of the most highly used R packages is "data.table" for efficient data reading/writing/manipulation/processing. Dr. Toby Hocking recommends that the students try to close some data.table issues, starting with those [labeled beginner-task](https://github.com/Rdatatable/data.table/labels/beginner-task) -- this is a really widely used R package, but their small dev team is not able to keep up with all the issues.

At the end of the term, it is expected that you have one or more contributions submitted (and hopefully reviewed by project members and accepted). Here are the overall steps and potential due dates.

### Final Report

Each group will write up a document describing the contribution process of an existing open source project, based on investigating the publicly available information about this project. This is the final deliverable of the project and can contain problems faced during the process, how you applied your knowledge, and what kind of new skills the group acquired during this journey. It needs to present an overview of the project and the task, followed by the details of your contribution process.
