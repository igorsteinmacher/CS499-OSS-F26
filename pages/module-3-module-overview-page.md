## **lightbulb.pngOverview**

This week we will explore some practices used in (Open Source) Software projects that are central to any project in the modern software development world.

More specifically, we will cover peer code review, continuous integration, and GitHub Actions. These practices are fundamental in maintaining the quality, reliability, and efficiency of open source projects.

**Peer Code Review**: This collaborative process involves developers examining each other’s code before it is merged into the main project. Peer code review not only helps catch bugs and improve code quality but also fosters a culture of knowledge sharing and collective ownership within the development team. We will explore best practices for conducting effective code reviews and discuss tools that facilitate this process.

**Continuous Integration (CI)**: CI is a development practice where code changes are automatically tested and merged into a shared repository multiple times a day. This ensures that the codebase remains in a deployable state and helps identify issues early in the development cycle. We will look at the principles of CI, its benefits, and how to set up a CI pipeline to automate testing and integration processes.

**GitHub Actions**: GitHub Actions is a powerful automation tool integrated into GitHub, enabling developers to create custom workflows for their projects. Whether it's automating code reviews, running tests, or deploying applications, GitHub Actions can streamline and enhance your development workflow. We will learn how to create and configure GitHub Actions to implement CI/CD pipelines and automate various tasks in your project.

- **Lectures:**

  - Continuous Integration/GitHub Actions
