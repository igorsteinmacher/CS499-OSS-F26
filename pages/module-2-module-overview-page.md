## **lightbulb.pngOverview**

This week we will explore the basic tools in modern software development. More specifically, we will cover source code versioning systems (git), and modern social coding platforms (GitHub).

Social coding platforms and code versioning systems are the core of contemporary software development practices. They facilitate collaboration, streamline workflows, and ensure that projects can be managed efficiently, regardless of team size or geographical distribution. Git, a distributed version control system, allows developers to track changes, revert to previous states, and branch out for feature development without disrupting the main codebase.

GitHub, a popular social coding platform built on top of Git, adds layers of collaboration through features like pull requests, issues, and project boards. These tools not only enhance code quality and project management but also foster a vibrant community of developers contributing to open-source projects worldwide.

Throughout this lecture, we will have the opportunity to go through the fundamental concepts and operations of Git, from commits to managing branches and resolving merge conflicts. We will also explore how GitHub enhances these functionalities with social coding aspects, enabling developers to collaborate seamlessly, review code, and contribute to a global shared knowledge and innovation ecosystem.

The Module includes:

- **Lectures:**
  - Working with git (from commits to conflict resolution)
  - Linking local and remote git
  - GitHub (issues, pull requests, and all)
- **Assignments covering:**
  - Using git
  - Basics of GitHub (issues, pull-requests)
  - Fork a repository and make a pull request
